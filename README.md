## Название и цель лабораторной работы

**Тема:** Разработка лексического анализатора для языка программирования Python (форматирование строк)

Спроектировать алгоритм (диаграмму состояний) и выполнить программную реализацию сканера для выделения лексем из входного текста. Интегрировать разработанный модуль в ранее созданный графический интерфейс языкового процессора.

   - учитывает многострочность входного текста.

## Вариант задания

**Номер варианта:** 49

**Описание варианта:**  
Разработать сканер для распознавания конструкции форматирования чисел в научную нотацию в языке Python: `f"{m:.Ne}"`

Данная конструкция представляет собой f-string с форматированием числа с плавающей точкой в экспоненциальном формате, где:
- `f` - префикс f-строки
- `m` - имя переменной (идентификатор)
- `.` - десятичная точка
- `N` - цифра или последовательность цифр, задающая точность
- `e` - символ экспоненты (научной нотации)

### Примеры корректных входных строк:

1. `f"{m:.2e}"` - форматирование с точностью 2 знака
2. `f"{m:.5e}"` - форматирование с точностью 5 знаков
3. `f"{m:.10e}"` - форматирование с точностью 10 знаков

### Допустимые лексемы:

| Тип лексемы | Описание | Условный код | Примеры |
|-------------|----------|--------------|---------|
| Ключевое слово | Префикс f-строки | 2 | `f` |
| Идентификатор | Имя переменной | 3 | `m` |
| Открывающая скобка | Символ `{` | 4 | `{` |
| Закрывающая скобка | Символ `}` | 5 | `}` |
| Открывающая кавычка | Символ `"` | 6 | `"` |
| Закрывающая кавычка | Символ `"` | 7 | `"` |
| Разделитель формата | Символ `:` | 8 | `:` |
| Десятичная точка | Символ `.` | 8 | `.` |
| Цифра | Цифра от 0 до 9 | 8 | `0-9` |
| Символ экспоненты | Символ `e` | 9 | `e` |

### Описание работы автомата

Конечный автомат работает следующим образом:

1. **Начальное состояние (start)** - ожидание первого символа 'f'
2. **Состояние 1** - распознан префикс 'f' (ключевое слово, код 2)
3. **Состояние 2** - распознана открывающая кавычка (код 6)
4. **Состояние 3** - распознана открывающая фигурная скобка '{' (код 4)
5. **Состояние 4** - распознан идентификатор 'm' (код 3)
6. **Состояние 5** - распознан разделитель ':' (код 8)
7. **Состояние 6** - распознана десятичная точка '.' (код 8)
8. **Состояние 7** - распознана цифра N (код 8) - может быть несколько цифр
9. **Состояние 8** - распознан символ экспоненты 'e' (код 9)
10. **Состояние 9** - распознана закрывающая фигурная скобка '}' (код 5)
11. **Состояние 10** - распознана закрывающая кавычка (код 7)

Если на любом этапе встречается символ, не соответствующий ожидаемому, автомат переходит в состояние ошибки.

## Интеграция с графическим интерфейсом

Разработанный сканер интегрирован в графический интерфейс языкового процессора, созданный в лабораторной работе №1.

### Функциональность интерфейса:

1. **Область ввода** - многострочное текстовое поле для ввода исходного кода
2. **Кнопка "Пуск"** - запускает лексический анализ
3. **Таблица результатов** (DataGridView) - отображает:
   - Условный код лексемы
   - Тип лексемы
   - Выделенную лексему
   - Местоположение (строка, позиции)
4. **Навигация по ошибкам** - при щелчке на строке с ошибкой курсор устанавливается на позицию недопустимого символа
